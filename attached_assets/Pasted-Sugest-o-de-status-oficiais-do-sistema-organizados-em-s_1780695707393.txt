4. Responsáveis por etapa

Vendedor / Televendas
•	Cadastrar o pedido no sistema.
•	Acompanhar pedidos em análise e pendências documentais.
•	Fazer interface com o cliente quando houver necessidade de documentos ou ajustes.
Financeiro
•	Analisar o pedido para aprovação ou reprovação.
•	Definir se o pedido será aprovado, negado ou ficará aguardando documentos.
Separação
•	Atuar nos pedidos que já foram produzidos e estão prontos para separação.
Faturamento
•	Emitir faturamento parcial ou total conforme o caso.
Transportadora / Logística
•	Assumir a etapa final após o faturamento e expedição do pedido.

6. Requisitos funcionais sugeridos para o software

•	Painel geral em formato de colunas (estilo Kanban) para visão macro da operação.
•	Painel individual por setor, mostrando apenas os pedidos relevantes para aquele usuário.
•	Filtros por cliente, vendedor, data, prioridade e status.
•	Alertas para pedidos parados há muito tempo.
•	Alerta para pendência de documentos.
•	Alerta para pedidos aguardando faturamento.
•	Registro de data e hora de cada mudança de status.
•	Histórico de movimentações do pedido.
•	Campo de observação por setor (comercial, financeiro, separação, faturamento e logística).
•	Possibilidade de gestores visualizarem todo o fluxo.
7. Fluxo operacional resumido

1.	Aguardando aprovação → análise do financeiro.
2.	Aprovado pelo financeiro → pedido liberado para produção.
3.	Reprovado / negado total → pedido encerrado.
4.	Reprovado / aguardando documentos → aguardando ação do cliente/comercial.
5.	Pedido enviado para produzir → etapa operacional em andamento.
6.	Pedido pronto aguardando separação → aguardando separação física.
7.	Pedido aguardando faturamento → pronto para emissão fiscal.
8.	Pedido faturado parcial → faturamento incompleto.
9.	Pedido faturado total → faturamento concluído.
10.	Pedido com a transportadora → pedido expedido.
8. Texto pronto para encaminhamento à empresa do software

Solicitamos o desenvolvimento de um fluxo de pedidos no sistema com etapas padronizadas, cores exclusivas e bem distintas entre si, além de visão segmentada por setor. O objetivo é garantir clareza operacional, facilitar o acompanhamento dos pedidos e permitir que cada usuário visualize prioritariamente apenas as etapas que exigem sua ação.

Status desejados: Aguardando aprovação; Aprovado pelo financeiro; Reprovado/negado total; Reprovado/aguardando documentos; Pedido enviado para produzir; Pedido pronto aguardando separação; Pedido aguardando faturamento; Pedido faturado parcial; Pedido faturado total; Pedido com a transportadora.

Também solicitamos filtros por setor, histórico de movimentações, alertas de pendências e painel geral para acompanhamento dos gestores.
9. Considerações finais
A padronização proposta visa reduzir erros, acelerar a leitura visual dos pedidos e tornar o fluxo mais intuitivo. A principal orientação deste documento é que as cores sejam nitidamente diferentes umas das outras, evitando confusão entre etapas próximas do processo.
